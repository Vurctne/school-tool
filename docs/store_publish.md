# Store publish guide — School Tool v2

Step-by-step reference for submitting School Tool to the Microsoft Store via Partner Center (Vurctne publisher account).

---

## Partner Center identity values

These three values are locked. They come from the Vurctne Partner Center account and must appear in `msix/AppxManifest.template.xml` and `app_metadata.py` exactly as shown — no normalisation, no reformatting.

| Field | Value |
|---|---|
| `Identity/@Name` | `Vurctne.VicSchoolFinanceTools` |
| `Identity/@Publisher` | `CN=E75204F6-F77B-4E0C-89C6-AC00A663F6A0` |
| `Properties/PublisherDisplayName` | `Vurctne` |

Source of truth in code: `app_metadata.py` constants `STORE_PACKAGE_IDENTITY_NAME`, `STORE_PUBLISHER`, `STORE_PUBLISHER_DISPLAY_NAME`.

## Build the Store upload artifact

```powershell
# From the repo root on Windows 11, with Windows 10 SDK on PATH or in .tools/msixsdk/
pwsh msix\build_msix_package.ps1 -StoreUpload
```

What `-StoreUpload` does differently from a plain build:

- Verifies the manifest `Identity/Name` and `Identity/Publisher` match the Partner Center values; fails loudly if they do not.
- Skips local `signtool.exe` signing (the Store re-signs after ingestion).
- Wraps the `.msix` in a `.msixupload` ZIP bundle that Partner Center requires.

Output files (both in `msix/output/`):

| File | Purpose |
|---|---|
| `School_Tool_2.0.0.0_x64.msix` | The raw package (local testing) |
| `School_Tool_2.0.0.0_x64.msixupload` | Upload to Partner Center |

SHA-256 checksum: `dist/SHA256SUMS-v2.0.0.txt`

## Pre-submission checklist

Work through this list before uploading to Partner Center. A failed certification wastes 24-48 h of review time.

- [ ] Icons present — run `python3 scripts/generate_store_icons.py` and confirm all 7 PNGs exist in `msix/staging/Assets/`.
- [ ] Manifest valid — open `msix/staging/AppxManifest.xml` (generated by the build, not the template) and confirm `Identity/@Name` and `Identity/@Publisher` match the Partner Center values above.
- [ ] Local install test — `Add-AppxPackage -Path msix\output\School_Tool_2.0.0.0_x64.msix` (requires Developer Mode or a trusted cert for sideload; use the unsigned build for this test).
- [ ] All three tools launch and produce output (HYIA, Master Budget Compass, Sub-Program Budget Report).
- [ ] Sub-Program Budget Report licence gate triggers correctly (no valid licence on the test machine).
- [ ] No crash on close.
- [ ] WACK (Windows App Certification Kit) passes — run from the Windows SDK: `appcert.exe runtest -apptype Desktop -testid c2ab18ba-... -packagefullname "Vurctne.VicSchoolFinanceTools_2.0.0.0_x64__<publisher hash>"`.
- [ ] Version is `2.0.0.0` (four-part; MSIX does not allow three-part).
- [ ] `APP_VERSION` in `app_metadata.py` matches the manifest version.
- [ ] `STORE_PUBLISHER` in `app_metadata.py` matches `Identity/@Publisher` in the manifest.

## Submit on Partner Center

1. Go to [partner.microsoft.com/dashboard](https://partner.microsoft.com/dashboard) and sign in as the Vurctne account owner.
2. Navigate to **Windows & Xbox > Apps and games > School Tool**.
3. Click **Start a submission** (or **New submission** if a draft exists).
4. Under **Packages**, upload `msix/output/School_Tool_2.0.0.0_x64.msixupload`.
5. Complete the Store listing (description, screenshots, category). See the separate listing-copy dispatch for the approved text.
6. Set pricing: Free (HYIA + Master Budget) with in-app purchase / licence gating for Sub-Program Budget Report.
7. Set age rating (IARC): General audience.
8. Set markets: Australia (primary); other English-speaking markets optional.
9. Submit for certification.

Certification typically takes 1-3 business days. Rejection reasons from WACK are shown in the Partner Center dashboard.

---

## Updating an existing submission (version bump)

1. Increment `APP_VERSION` in `app_metadata.py` (e.g. `2.0.1`).
2. Rerun `pwsh msix\build_msix_package.ps1 -StoreUpload`.
3. Submit the new `.msixupload` as a new submission in Partner Center.

The Store auto-updates installed copies on enrolled devices within 24 h of certification completion.

## Sideload build (developer testing, not Store)

```powershell
# Unsigned — Developer Mode must be on
pwsh msix\build_msix_package.ps1 -NoSign

# Self-signed (requires a PFX and the cert trusted on the test machine)
$env:SFT_SIGN_PFX = "C:\certs\dev-cert.pfx"
$env:SFT_SIGN_PW  = "password"
pwsh msix\build_msix_package.ps1
```

## Notes

- The `.msixupload` format is a ZIP renamed with the `.msixupload` extension. Do not try to open it as a regular ZIP — Partner Center handles extraction.
- The build script auto-discovers `makeappx.exe` from `.tools/msixsdk/`, then `PATH`, then Windows SDK installations under `%ProgramFiles(x86)%\Windows Kits\10\bin\`.
- `Square310x310Logo.png` was added in Round 16. Run `python3 scripts/generate_store_icons.py` once after pulling to regenerate all icons including this new size.
